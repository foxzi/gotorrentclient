# Веб-интерфейс (RU)

Веб-режим запускает gotorrentclient как демон со встроенным torrent-движком, который качает несколько торрентов одновременно, и управляет ими через браузер.

## Запуск

```bash
./gotorrentclient --web --username admin --password secret --download-dir ./downloads
```

По умолчанию сервер слушает `:8080`. Откройте `http://localhost:8080` и войдите.

Если не указать `--username` / `--password`, сервер запустится без аутентификации —
интерфейс будет доступен любому в сети. В stderr выводится предупреждение.

## Переменные окружения

Имеют приоритет над одноимёнными флагами:

| Переменная | Назначение | По умолчанию |
|------------|------------|--------------|
| `GTC_LISTEN` | Адрес прослушивания | `:8080` |
| `GTC_USERNAME` | Логин | — |
| `GTC_PASSWORD` | Пароль | — |

## Флаги движка

Задаются при старте демона и действуют на все торренты:

| Флаг | Назначение |
|------|------------|
| `--download-dir` | Каталог загрузок |
| `--max-peers` | Макс. число пиров на торрент |
| `--download-rate` | Лимит скачивания, Мбит/с (0 — без лимита) |
| `--upload-rate` | Лимит отдачи, Мбит/с (0 — без лимита) |
| `--enable-seeding` | Раздавать после завершения |
| `--seed-ratio` | Целевой рейтинг раздачи |
| `--proxy` | Прокси (http, https, socks5) |

## Авторизация

Один пользователь из конфигурации. После входа выдаётся cookie-сессия (HttpOnly).
Все страницы, кроме входа и статики, требуют авторизации. Кнопка выхода завершает сессию.

## Интерфейс

**Добавление торрентов**

- Вставьте magnet-ссылку в текстовое поле и нажмите **Add**.
- Загрузите один или несколько файлов `.torrent` через файловый выборщик.
- Оба варианта можно отправить одной формой. Ошибки отображаются без перезагрузки страницы.

**Колонки таблицы торрентов**

| Колонка | Описание |
|---------|----------|
| Name | Название торрента |
| Progress | Процент завершения |
| Downloaded | Скачано / общий размер |
| Down | Текущая скорость скачивания |
| Up | Текущая скорость отдачи |
| Peers | Число активных пиров |
| Uploaded | Всего отдано |
| Status | `Metadata` / `Downloading` / `Checking` / `Paused` / `Done` |

**Действия с торрентом**

| Кнопка | Действие |
|--------|----------|
| Pause | Остановить передачу данных без удаления торрента |
| Resume | Возобновить передачу данных |
| Verify | Перепроверить хэши локальных файлов |
| Drop | Удалить торрент (загруженные файлы остаются) |

## Персистентность

Состояние торрентов сохраняется автоматически в директорию `.gotorrentclient/` внутри
каталога загрузок. При следующем запуске все ранее добавленные торренты восстанавливаются,
включая их статус (активен / на паузе).

## REST API

Веб-сервер предоставляет JSON API, используемый браузерным интерфейсом. Все запросы
требуют авторизации, если учётные данные заданы.

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/torrents` | Список всех торрентов |
| POST | `/api/add` | Добавить magnet или загрузить `.torrent` файл(ы) |
| POST | `/api/drop` | Удалить торрент (поле формы `id`) |
| POST | `/api/pause` | Поставить на паузу (поле формы `id`) |
| POST | `/api/resume` | Возобновить (поле формы `id`) |
| POST | `/api/verify` | Проверить данные (поле формы `id`) |

При успехе возвращается `{"torrents": [...]}`, при ошибке — `{"error": "...", "torrents": [...]}`.
